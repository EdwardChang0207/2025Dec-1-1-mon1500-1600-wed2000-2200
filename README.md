## 🗺️ 課程規劃地圖

### 🚀 第一階段：掌握程式邏輯的第一步 (上篇)

- [ ] **Session 1：變數與資料型態 (Variables & Data Types)**
    - 變數命名規則、記憶體與標籤概念
    - 基礎型態：整數 (int)、浮點數 (float)、布林值 (bool)
    - 型態查詢與轉換：`type()`, `int()`, `str()`
    - 輸出輸入互動：`print()` 與 `input()`

- [ ] **Session 2：運算子與序列初探 (Operators & Sequences Intro)**
    - 算術運算子：加減乘除、餘數 (%)、整除 (//)、次方 (**)
    - 賦值運算子：累加 (+=)、累減 (-=)
    - 序列概念：字串不只是文字，更是「字元的序列」
    - 基礎操作：索引 (Index)、字串相加與重複

- [ ] **Session 3：流程控制 - 判斷與迴圈 (Flow Control)**
    - 邏輯分歧：`if`, `else`, `elif` 與比較運算子
    - 邏輯運算：`and`, `or`, `not` 的組合運用
    - 重複執行：For 迴圈 (`range` 控制) 與 While 迴圈
    - 流程中斷：`break` (跳出) 與 `continue` (略過)

- [ ] **Session 4：序列深究 (1) - 串列 (Lists)**
    - 串列建立：儲存多個資料的容器 `[]`
    - 通用操作：切片 (Slicing) `[start:end:step]`
    - 專屬方法：`append`, `insert`, `remove`, `pop`
    - 串列運算：`sort`, `reverse`, `len`

- [ ] **Session 5：序列深究 (2) - 元組 (Tuples)**
    - 元組特性：`()` 與「不可變動 (Immutable)」的意義
    - Tuple vs List：資料保護 vs 資料操作
    - 拆包 (Unpacking)：`x, y = point`
    - 序列總結：字串、串列、元組的異同比較

### 🌟 第二階段：向進階開發邁進 (下篇)

- [ ] **Session 6：鍵值對應 - 字典 (Dictionaries)**
    - 字典結構：鍵 (Key) 與值 (Value) 的對應 `{key: value}`
    - 資料存取：新增、修改與查詢
    - 字典走訪：`keys()`, `values()`, `items()`
    - 應用場景：儲存結構化資料

- [ ] **Session 7：模組化編程 - 函數 (Functions)**
    - 定義與呼叫：`def` 關鍵字，邏輯封裝
    - 參數傳遞：必填參數、預設參數
    - 回傳值：`return` 機制與多值回傳
    - 變數作用域：區域 (Local) 與 全域 (Global) 的區別

- [ ] **Session 8：物件導向入門 (Objects)**
    - OOP 概念：類別 (Class) vs 物件 (Object)
    - 定義類別：`class` 關鍵字
    - 屬性與建構子：`__init__` 與 `self` 的意義
    - 方法 (Methods)：物件內的行為

- [ ] **Session 9：錯誤管理與例外 (Exception Handling)**
    - 常見錯誤：`SyntaxError`, `TypeError`, `IndexError`, `KeyError`
    - 例外捕捉：`try...except` 結構
    - 進階處理：`else` 與 `finally`
    - 程式穩健性：預防崩潰的設計思維

- [ ] **Session 10：資料結構與演算法 (DS & Algorithms)**
    - 演算法基礎：時間複雜度與空間複雜度的直觀概念
    - 搜尋演算法：線性搜尋 (Linear) vs 二分搜尋 (Binary)
    - 排序演算法：泡沫排序 (Bubble Sort) 的原理與實作
    - 綜合應用：如何根據資料量選擇正確的工具

## 💡 助教的小叮嚀
1.  **動手打程式碼**：看懂不代表會寫，親手把範例打一遍是最好的學習方式。
2.  **不要怕報錯 (Error)**：紅色的錯誤訊息是程式設計師最好的朋友，它會告訴你哪裡需要修正。
3.  **保持好奇心**：隨時可以問老師「為什麼這裡要這樣寫？」或是「有沒有別的方法？」。
